# 卫星脚本入口说明（新组织）

本目录新增“每卫星独立入口 + 公共模式模块”组织：

- 每卫星入口：`p2p_sat_<SAT>.csh`
- 公共路由：`p2p_sat_entry.csh`
- 公共模式：`p2p_mode_strip.csh` / `p2p_mode_tops.csh`

## 1. 当前入口脚本

- strip：`ERS ENVI ALOS ALOS_SLC ALOS2 ALOS2_SCAN S1_STRIP ENVI_SLC CSK_RAW CSK_SLC CSG TSX RS2 GF3 LT1 DJ1`
- tops：`S1_TOPS`

## 2. 使用方式

```bash
# 以 RS2 为例（strip）
p2p_sat_RS2.csh <master> <aligned> [config]

# 以 S1_TOPS 为例（tops）
p2p_sat_S1_TOPS.csh <master_scene> <aligned_scene> [config]
```

内部会转发到 `p2p_processing.csh`，因此保留现有处理行为。

## 3. 新卫星接入建议

1. 先判断模式：`strip` 或 `tops`。
2. 新建入口脚本：`p2p_sat_<NEW_SAT>.csh`。
3. 若有独立处理需求，优先通过显式钩子脚本扩展：
   - `p2p_hook_stage2.csh`
   - `p2p_hook_stage3.csh`
   - `p2p_hook_stage4.csh`
4. 通过卫星专用插件脚本注入（可选）：
   - `p2p_hook_stage2_<SAT>.csh`
   - `p2p_hook_stage3_<SAT>.csh`
   - `p2p_hook_stage4_<SAT>.csh`
5. 用 `tools/stage2_only_minimal_check.sh` 先做 L1 预检。

## 4. Hook 状态码约定

- `pre` 钩子返回：
  - `0`：继续执行默认阶段逻辑
  - `10`：跳过默认阶段逻辑
  - `20`：插件已自行处理该阶段逻辑
  - 其他：视为错误并退出
- `post` 钩子返回：
  - `0`：成功
  - 其他：视为错误并退出

详细开发说明见：`卫星Hook开发说明.md`。
