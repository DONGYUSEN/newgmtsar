# SAR 工具函数模块总结

## 目录结构

```
utils/
├── sar_utils.py         # 核心工具函数模块
├── snaphu_utils.py      # snaphu 解缠稳健执行工具（重试/版本检查/掩膜与tile策略）
├── geosar.py           # 科研级 DEM -> SAR 模拟 + 映射表输出
├── ll2sar.py           # 经纬度/UTM -> SAR 坐标转换
├── sar2ll.py           # SAR 坐标 -> 经纬度/UTM 转换
├── mkdem.py            # DEM 处理模块
├── multilook.py        # 多视处理模块
├── dj1_l1_processor.py # 天仪SAR L1数据处理器
├── old/                # 旧版本文件
└── old2/               # 旧版本文件
```

## 文件功能分析

### 1. sar_utils.py

**核心工具函数模块**，提供 SAR 处理中常用的基础工具函数。

**主要功能：**
- 坐标转换：
  - `llh_to_xyz`：经纬度高程转换为地心坐标
  - `xyz_to_llh`：地心坐标转换为经纬度高程
  - `llh_to_xyz_vectorized`：向量化版本的坐标转换
  - `xyz_to_llh_vectorized`：向量化版本的坐标转换

- 轨道插值：
  - `orbit_hermite`：4点Hermite多项式轨道插值
  - `interpolate_wgs84_orbit`：WGS84轨道插值
  - `interpolate_sch_orbit`：SCH轨道插值
  - `linear_interpolation`：线性轨道插值
  - `calculate_orbit_position`：计算指定时间的卫星位置和速度

- 多普勒计算：
  - `calculate_doppler`：计算多普勒频率和导数
  - `load_doppler_parameters`：从YAML文件加载多普勒参数

- 图像处理：
  - `read_image`：读取图像文件
  - `read_yaml`：读取YAML文件
  - `write_yaml`：写入YAML文件
  - `generate_offset_field`：生成完整的多项式偏移场

- 相关性计算：
  - `compute_correlation`：计算相关系数
  - `compute_snr`：计算信噪比
  - `compute_rmse`：计算均方根误差
  - `compute_local_correlation`：计算局部相关系数

- 辅助函数：
  - `_i0`：零阶修正贝塞尔函数
  - `_kaiser_window`：生成kaiser窗口
  - `calculate_look_direction`：计算卫星视线方向
  - `get_sar_image_corners`：从YAML文件获取SAR图像四个角点的坐标

**性能优化：**
- 集成了Numba加速，对核心函数进行了JIT编译
- 支持并行处理，提高大规模数据处理效率

### 2. geosar.py

**科研级 DEM -> SAR 模拟 + 映射表输出**，是整个SAR处理流程的核心引擎。

**主要功能：**
- DEM 到 SAR 坐标转换（geo2rdr）
- SAR 到 DEM 坐标转换（rdr2geo）
- 零-多普勒时间计算
- 轨道模拟与插值
- 阴影/叠置检测
- HDF5 格式输出，包含映射表和SAR产物

**技术特点：**
- 使用Numba和多进程处理提高性能
- 支持真实轨道数据和模拟轨道
- 向量化计算，减少Python循环开销

### 3. ll2sar.py

**经纬度/UTM/投影栅格 -> SAR 栅格**，提供点查询和栅格转换功能。

**主要功能：**
- 点查询：将地理坐标（经纬度或UTM）转换为SAR像素坐标
- 栅格转换：将地理参考的GeoTIFF重采样到SAR网格
- 支持双线性插值和Lanczos resampling

**技术特点：**
- 避免使用KDTree和griddata，使用局部双线性近似
- 支持复数数据的sinc插值
- 分块处理，减少内存使用

### 4. sar2ll.py

**SAR 坐标系数据重采样到地理坐标系**，支持经纬度或UTM输出。

**主要功能：**
- SAR 数据到地理坐标的转换
- 支持多种输入格式（GeoTIFF、.npy、HDF）
- 支持复数数据的sinc插值
- 自动计算UTM分带

**技术特点：**
- 基于DEM->SAR映射表进行反向转换
- 分块处理，支持大型SAR图像
- 自动检测并使用HDF中的lat/lon网格

### 5. mkdem.py

**DEM处理模块**，提供DEM下载、裁剪、重投影功能。

**主要功能：**
- 从ESA STEP镜像下载SRTMGL1数据
- 对DEM进行裁剪和重采样
- 输出经纬度和UTM两种投影的GeoTIFF和VRT
- 支持填充DEM中的空洞

**技术特点：**
- 最小依赖设计，避免引入重依赖
- 支持从master.yaml的角点范围推算DEM覆盖范围
- 自动处理SRTM瓦片的拼接

### 6. multilook.py

**多视处理模块**，对SAR图像进行多视处理，降低分辨率，提高信噪比。

**主要功能：**
- 对SAR数据进行多视处理
- 支持复数数据的相位保持多视
- 分块处理，避免内存溢出
- 更新YAML文件参数以反映多视处理后的变化

**技术特点：**
- 参考ISCE2-2.6.4中的multilook算法实现
- 支持多种边界处理方式
- 提供详细的统计信息

### 7. dj1_l1_processor.py

**天仪SAR L1数据处理器**，处理天仪系列SAR L1数据，生成SLC数据文件、轨道文件和GDAL VRT文件。

**主要功能：**
- 解析XML元数据文件
- 提取轨道数据
- 生成GDAL VRT文件
- 生成YAML元数据文件

**技术特点：**
- 支持从XML文件提取完整的元数据和轨道信息
- 生成符合GMTSAR格式的YAML文件
- 自动处理时间格式转换

### 8. snaphu_utils.py

**snaphu 解缠稳健执行工具**，用于统一管理 snaphu 的输入规范、参数与失败回退策略。

**主要功能：**
- 输入规范化：将 `FLOAT_DATA` 模式输入相位统一到 `[0, 2π)`，并清洗 NaN/Inf
- 质量掩膜：基于相干性阈值与有效像素生成 `BYTEMASK`（`-M`）
- 分块策略：支持显式 tile 参数和按影像规模自动分块
- 稳定执行：多级回退（并行 tile -> 串行 tile -> 非 tile -> `mcf` 回退 `mst` -> 取消掩膜）
- 诊断输出：记录每次尝试的命令/日志，失败时给出汇总信息
- 版本告警：检测 snaphu 版本，对 `< 2.0.7` 给出官方已知问题提醒

## 模块间关系

```
+-------------------+      +-------------------+      +-------------------+
|   sar_utils.py    |<-----|    geosar.py      |<-----|  dj1_l1_processor.py |
+-------------------+      +-------------------+      +-------------------+
        ^                          |                          |
        |                          |                          |
        |                          v                          |
        |                  +-------------------+              |
        +------------------|    ll2sar.py      |<-------------+
        |                  +-------------------+              |
        |                          ^                          |
        |                          |                          |
        v                          |                          |
+-------------------+      +-------------------+              |
|   multilook.py    |      |    sar2ll.py      |<-------------+
+-------------------+      +-------------------+              |
                                ^                          |
                                |                          |
                                v                          |
                        +-------------------+              |
                        |    mkdem.py       |<-------------+
                        +-------------------+
```

**核心依赖关系：**
1. **sar_utils.py** 是基础工具库，被其他所有模块依赖
2. **geosar.py** 是核心引擎，依赖sar_utils.py，同时被ll2sar.py和sar2ll.py依赖
3. **ll2sar.py** 和 **sar2ll.py** 互为逆操作，都依赖geosar.py的输出
4. **mkdem.py** 独立处理DEM数据，但其输出会被geosar.py使用
5. **multilook.py** 处理SAR图像，可独立运行，也可与其他模块配合使用
6. **dj1_l1_processor.py** 处理原始SAR数据，生成的文件可被其他模块使用

## 应用场景

1. **SAR 数据处理流程**：
   - 使用dj1_l1_processor.py处理原始L1数据
   - 使用mkdem.py准备DEM数据
   - 使用geosar.py进行DEM到SAR的转换
   - 使用multilook.py进行多视处理
   - 使用sar2ll.py进行地理编码

2. **坐标转换**：
   - 使用ll2sar.py将地理坐标转换为SAR坐标
   - 使用sar2ll.py将SAR坐标转换为地理坐标

3. **精度评估**：
   - 使用sar_utils.py中的相关性计算函数评估配准精度

## 性能优化建议

1. **内存管理**：
   - 对于大型SAR图像，使用分块处理避免内存溢出
   - 合理设置缓存大小，平衡内存使用和计算速度

2. **并行处理**：
   - 充分利用Numba的并行处理能力
   - 对于IO密集型操作，使用多进程处理

3. **算法优化**：
   - 对于频繁调用的函数，使用Numba JIT编译
   - 对于向量化操作，充分利用NumPy的向量化能力

4. **数据格式**：
   - 使用HDF5存储大型数据集，提高读写效率
   - 对于中间结果，考虑使用内存映射文件

## 总结

utils目录包含了一套完整的SAR数据处理工具，从原始数据处理、DEM准备、坐标转换到图像后处理，形成了一个完整的处理流程。各模块之间职责明确，接口清晰，相互配合形成了一个功能强大的SAR处理系统。

核心优势：
- 模块化设计，便于维护和扩展
- 性能优化，支持大规模数据处理
- 功能完整，覆盖SAR处理的主要环节
- 接口友好，易于集成到其他系统

未来发展方向：
