# 基于真实数据的单对 DInSAR 初步大气误差矫正研究思路

## 1. 目标
针对单组（单干涉对）DInSAR结果，结合GMTSAR数据组织方式与真实数据（`~/Temp/yaxia`），构建一条可执行的初步大气改正流程，输出：

- 大气矫正前干涉图（相位）
- 大气矫正后干涉图（相位）

## 2. 数据与输入
- SAR/GMTSAR相关输入（真实）：
  - `~/Temp/yaxia/SLC/*.PRM`（轨道与成像参数）
  - `~/Temp/yaxia/SLC/*.SLC`（SLC数据）
  - `~/Temp/yaxia/topo/dem.grd`（DEM）
- 已有干涉结果（真实）：
  - `~/Temp/yaxia/test/interferogram/filt_topophase.unw.geo(.vrt)`（解缠相位）
  - `~/Temp/yaxia/test/interferogram/phsig.cor.geo(.vrt)`（相干系数）
  - `~/Temp/yaxia/test/geometry/los.rdr.geo(.vrt)`（视线几何）

说明：当前目录下暂无现成高分辨率气象3D场（P/T/q）或同轨道时刻ZTD栅格，因此先实现“DEM高程分层+可选外部气象延迟场”的初步改正。

## 3. 方法框架

### 3.1 干涉相位准备
读取解缠相位 `phi_unw` 与相干系数 `coh`，并构建稳定像元掩膜（如 `coh >= 0.35`）。

### 3.2 DEM重采样与高程相关项建模
将 `dem.grd` 重采样到干涉图网格后，采用稳健线性模型拟合高程分层项：

\[
\phi_{atm,elev} = a\,h + b
\]

用迭代剔除离群值（sigma-clipping）提升稳健性，降低局部形变/噪声影响。

### 3.3 可选：外部气象延迟场注入
若后续提供主辅影像时刻的大气延迟（如ZTD）栅格：

\[
\Delta L_{atm}=L_{slave}-L_{master}
\]

基于LOS入射角投影并转相位：

\[
\phi_{atm,meteo} = -\frac{4\pi}{\lambda}\frac{\Delta L_{atm}}{\cos\theta}
\]

其中 \(\lambda\) 来自PRM（或配置），\(\theta\) 来自LOS几何。

### 3.4 相位改正与产品输出

a) 解缠相位改正：
\[
\phi_{corr} = \phi_{unw} - \phi_{atm,elev} - \phi_{atm,meteo}
\]

b) 为可视化输出缠绕相位（[-π, π]）：
\[
\phi_{wrap}=\arg(e^{i\phi})
\]

输出GeoTIFF与PNG：
- `before_wrapped_phase.*`
- `after_wrapped_phase.*`
- `atm_elevation_phase.*`（诊断）

## 4. 与GMTSAR的结合点
- 继承GMTSAR数据组织与参数（PRM、SLC、topo）。
- 脚本中保留GMTSAR干涉生成步骤占位（可按需启用`intf.csh`重跑干涉对）。
- 当前优先用`~/Temp/yaxia/test`已有真实干涉成果完成快速闭环，保证先产出“前后对比图”。

## 5. 验证指标
- 改正前后相位标准差对比（稳定区）。
- 相位-高程相关系数下降情况（|r|减小）。
- 视觉检查：山区条带状分层信号是否减弱。

## 6. 风险与后续增强
- 单干涉对无法彻底分离形变与湍流大气，仅为初步改正。
- 后续建议接入同化气象产品（ERA5/GACOS等）逐时刻延迟场，替代或补强线性高程项。
- 可加入空间低通/时空约束（多时相）进一步稳健化。

## 7. 当前目录执行方式（已落地脚本）
- 脚本：`./gmtsar_atm_workflow.sh`
- 默认输入：`~/Temp/yaxia`下真实数据（含`test/interferogram`与`topo/dem.grd`）
- 默认行为：先用已有真实干涉结果做改正闭环；GMTSAR重算步骤默认关闭（`RUN_GMTSAR_INTF=0`）

运行示例：

```bash
cd /home/ysdong/Software/GMTSAR/atom
./gmtsar_atm_workflow.sh
```

如需接入外部大气延迟栅格（主/辅时刻）：

```bash
ATM_MASTER=/path/to/master_ztd.tif \
ATM_SLAVE=/path/to/slave_ztd.tif \
ATM_SCALE=1.0 \
./gmtsar_atm_workflow.sh
```

输出目录：`./results/`
- `before_wrapped_phase.png` / `.tif`
- `after_wrapped_phase.png` / `.tif`
- `atm_elevation_phase.png` / `.tif`
- `metrics.json`
